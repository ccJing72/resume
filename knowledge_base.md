# Knowledge Base

## Resume Variants

记录时间：2026-06-21

当前目录下维护三套简历，它们是同一份职业经历的不同投递版本，后续修改时要区分定位，不要互相覆盖。

### 1. 原始综合版

- 数据源：`resume.md`
- HTML 成品：`景成成_Java后端架构师.html`
- 预览图：`景成成_Java后端架构师_preview.png`
- 定位：综合型 Java 后端架构师简历，兼顾平台型产品建设、能碳管理、AI Agent、智慧医院后勤、数字政务等经历。
- 维护规则：这是用户最初提供的原始简历版本。除非用户明确要求修改原始简历，否则不要改动这两个原始文件。

### 2. AI Agent 落地方向

- 数据源：`resume_ai_agent.md`
- HTML 成品：`景成成_AI_Agent落地.html`
- 定位：AI Agent 落地工程师 / AI 应用后端架构师 / Java + AI 工程化方向。
- 内容重点：突出 EnerGraph、LangGraph、ReAct、RAG、知识库问答、业务 API 工具调用、FastAPI SSE、UIAction 页面跳转、低置信拒答、引用来源标注、Token 自动刷新，以及 AI 与真实业务系统集成。
- 取舍原则：能源管理、碳管理、数据建模等内容作为 AI Agent 的业务底座呈现；早期政务和传统 Java 项目内容保持简洁。

### 3. Java 后端架构师 / 技术负责人方向

- 数据源：`resume_java_architect.md`
- HTML 成品：`景成成_Java后端架构师_技术负责人.html`
- 定位：Java 后端架构师 / 技术负责人 / 高级 Java 后端开发工程师。
- 内容重点：突出平台架构、技术选型、核心模块研发、数据建模、OneData / OneService、权限体系、消息与告警、时序数据、自动化部署、高可用改造、团队协作和交付管理。
- 取舍原则：AI Agent 经历保留为加分项，但不抢主线；整体主线是 Java 后端架构能力和技术负责人经验。

## Maintenance Rules

- 需要改某一方向简历时，同步修改该方向的 Markdown 数据源和 HTML 成品。
- 原始综合版、新 AI Agent 版、新 Java 架构/技术负责人版是并列版本，不是覆盖关系。
- 用户如果说“这三套简历”，通常指：
  1. `resume.md` + `景成成_Java后端架构师.html`
  2. `resume_ai_agent.md` + `景成成_AI_Agent落地.html`
  3. `resume_java_architect.md` + `景成成_Java后端架构师_技术负责人.html`
- 生成 PDF 或预览时优先从对应 HTML 打开并导出。
